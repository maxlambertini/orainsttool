Oracle Instance Script Generation

This page is a quick tool to help DBA create a database instance on linux systems when X11 -- and thus dbca -- is not available. This tools is opinionated, that is it enforces a datafile tree which may or may not be of your liking. If you don't like it, just change directories generated in #3 and double-check the path specified in CREATE DATABASE section (#5).

A note to the DBA: the DB created is in NOARCHIVELOG mode.

Instructions

Fill the input boxes with the appropriate values. Then, follow the steps, copy the preformatted code and paste it in a unix shell. For more informations, refer to this link from Oracle to learn how to create an Oracle instance using the command line.

1. Setup environment variables

set %ORACLE_HOME%= set %ORACLE_SID%= set PATH=%PATH%;

2. Create password file and init.ora

REM create password file orapwd FILE=\orapw password= entries=30 FORCE=Y REM create init.ora ( ) > \init.ora EOF REM windows only: oradim -NEW -SID -STARTMODE MANUAL -PFILE \init.ora

3. Create data directories

mkdir mkdir mkdir mkdir mkdir mkdir mkdir # create paths for redo mkdir # create paths for ctl mkdir

4. Boot instance and create spfile

( echo connect / as sysdba echo create spfile from pfile='\init.ora'; echo startup nomount; echo exit; echo ) | sqlplus /nolog

5. CREATE DATABASE!

( echo connect / as sysdba echo echo CREATE DATABASE echo USER SYS IDENTIFIED BY echo USER SYSTEM IDENTIFIED BY echo LOGFILE echo MAXLOGFILES 20 echo MAXLOGMEMBERS 5 echo MAXLOGHISTORY 1 echo MAXDATAFILES 100 echo CHARACTER SET US7ASCII echo NATIONAL CHARACTER SET AL16UTF16 echo EXTENT MANAGEMENT LOCAL echo DATAFILE '/system01.dbf' SIZE 325M REUSE echo SYSAUX DATAFILE '/sysaux' SIZE 325M REUSE echo DEFAULT TABLESPACE users echo DATAFILE '/users01.dbf' echo SIZE 500M REUSE AUTOEXTEND ON MAXSIZE UNLIMITED echo DEFAULT TEMPORARY TABLESPACE temp echo TEMPFILE '/temp01.dbf' echo SIZE 20M REUSE echo UNDO TABLESPACE undotbs echo DATAFILE '/undotbs01.dbf' echo SIZE 200M REUSE AUTOEXTEND ON MAXSIZE UNLIMITED; echo echo exit; ) | sqlplus /nolog ( echo connect / as sysdba echo echo @?/rdbms/admin/catalog.sql echo @?/rdbms/admin/catproc.sql echo exit; ) | sqlplus /nolog ( echo connect system/ echo echo @?/sqlplus/admin/pupbld.sql echo echo exit; ) | sqlplus /nolog REM then, you can enable instance startup oradim -EDIT -SID -STARTMODE AUTO -SRVCSTART SYSTEM -SPFILE
